## How to extend this audit to actual CWV numbers

The GSC API doesn't return CWV field metrics. Two practical options:

1. **Export from GSC UI** — Search Console → Experience → Core Web Vitals, click "Open Report" for Mobile and Desktop, export the failing/needs-improvement URL groups as CSV. Drop the file in the repo and rerun this script with a parser.
2. **Add a `psi`/`crux` handler to the edge function** — set a `GOOGLE_API_KEY` env var on the Supabase project (CrUX + PSI both work with simple API keys, no OAuth scope required), then deploy a small handler that proxies `pagespeedonline/v5` and `chromeuxreport/v1`. The script can then loop the URLs above and write LCP / INP / CLS p75 + Lighthouse opportunity audits per URL.
